
## 概要
メソッド内で宣言される変数等の使い捨ての変数はGCの対象になりGCスパイクを誘引し、結果としてVR酔いを引き起こす恐れがあります

## 説明

float型やVector形式のデータはメモリ内のスタック領域に確保されるため、ゲームの処理と関係のない場所でGCの対象になりGCスパイクとして現れることはありませんが、自作のクラスをメソッド内で宣言して利用した場合等はヒープ領域にメモリの確保を行うため結果としてGCスパイクが生じます。

## 対策

使い捨てになる前提の変数はクラス宣言時に宣言しておき、利用する際に代入する事（キャッシュ）によってGCの対象になる事を回避できます。

```csharp
public class MyComponent : MonoBehaviour {
	// このように事前に宣言しておく事でMyComponentがGCされるまでこの変数もGCされません
	private MyClass _cachedClass;
	
	private void Update (){
		_cachedClass = new MyClass();
	}
}
```

## 補足

スタック領域に確保される型の例は以下のとおりです。

- **プリミティブ型**: `int`, `float`, `double`, `bool`, `byte`, `char` など。
    
- **列挙型（Enum）**: 内部的には整数として扱われるため、すべて値型です。
    
- **値タプル（ValueTuple）**: `(int, float)` のようなC# 7.0以降のタプル。
    
    - _注意_: 古い `System.Tuple<T1, T2>` はクラス（参照型）なのでヒープに確保されます。必ず `System.ValueTuple`（括弧による構文）を使用してください。
        
- **カスタム構造体**: 自身で `struct` キーワードを用いて定義した型（`public struct PlayerData { ... }`）。
  
- **ベクトル・行列**: `Vector2`, `Vector3`, `Vector4`, `Quaternion`, `Matrix4x4`
    
- **色情報**: `Color`, `Color32`
    
- **物理判定関連**: `Ray`, `RaycastHit`, `RaycastHit2D`
    
- **領域・矩形**: `Bounds`, `Rect`, `RectInt`, `Vector2Int`, `Vector3Int`
  
これらの型を利用した場合でも配列やリストにした場合はヒープ領域に確保されるため、配列やリストはキャッシュ領域を定義する事が望ましいです。
