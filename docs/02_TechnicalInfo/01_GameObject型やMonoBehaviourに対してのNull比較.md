## 概要

GameObject型やMonoBehaviourに対してNullチェックを行う行為を頻繁に行うのは控えたほうが良い。

## 説明

UnityにおけるGameObject型とMonoBehaviourはUnityエンジン本体、もっと踏み込んでいうのであればC++のレイヤーで管理されており、C#で書いた我々のコードからそれらのNullを取得する事はネイティブのC#におけるnull比較よりも有意に高い計算コストを要します。

そのため、毎フレームそのような処理を行う際には以下のような迂回策が望ましいです

- is 比較演算子による簡易比較
```csharp
//このような比較処理が望ましい
if (gameObject is null){
}
```
