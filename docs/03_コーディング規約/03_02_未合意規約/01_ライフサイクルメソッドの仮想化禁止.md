
## 概要

StartやUpdate、Awakeを直接仮想メソッドにする事を禁止する

## 理由

- 基底クラスで記述した重要な処理の実行を後々の派生クラスで保証できないため
- base.Start()などを利用した継承元の処理の実行では処理順序のコントロールが複雑になり、結果として意図しない処理が発生する恐れがあるため

## 代替策

### ライフサイクルメソッドのラッパーメソッドを定義
```csharp
private void Start () {
	PreStart();
	.....
	PostStart();
}

// 継承先では以下のメソッドをoverrideして新しい挙動を実装する

protected virtual void PreStart () {}

protected virtual void PostStart () {}
```
